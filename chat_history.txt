1. Docker stack stabilized (frontend + backend + postgres + nginx)

2. Auth routing issue fixed

3) AGREEMENT ABOUT PRODUCT IMAGES

Where to put images
- public/products/

Naming rules
- lowercase letters + numbers + hyphen only
- no spaces, no cyrillic, no special symbols
- base image: slug.webp
- extra images: slug-2.webp, slug-3.webp

Recommended format and size
- Prefer: webp
- Also acceptable: jpg/png
- Recommended size: 1200x1200 (1:1)
- Recommended file size: <= 300-400 KB each

How product metadata is provided
- Do NOT put price/rating in filename.
- Metadata is provided separately as text list.

Minimal input from user per product
- slug
- name
- price

Fields that can be auto-filled by assistant if needed
- rating
- category
- inStock
- description
- specs

Example line format for user-provided list
- slug | Product Name | 350000
