# Fail2Ban setup for VOLT Store

This directory contains ready-to-use Fail2Ban filters for Nginx logs generated by Docker.

## What is included

- `filter.d/nginx-auth-bruteforce.conf` for repeated failed auth attempts
- `filter.d/nginx-scanners.conf` for scanner/probe traffic
- `jail.local.example` with safe defaults

## Prerequisites

- Linux host with `fail2ban` package installed
- Project deployed with Docker Compose
- Nginx service running with log mount:
  - `./logs/nginx:/var/log/nginx`

## Install on host (Ubuntu/Debian)

```bash
sudo apt update
sudo apt install -y fail2ban
```

## Copy configs

Replace `/opt/volt_store` with your real repository path.

```bash
sudo mkdir -p /etc/fail2ban/filter.d
sudo cp /opt/volt_store/security/fail2ban/filter.d/nginx-auth-bruteforce.conf /etc/fail2ban/filter.d/
sudo cp /opt/volt_store/security/fail2ban/filter.d/nginx-scanners.conf /etc/fail2ban/filter.d/
sudo cp /opt/volt_store/security/fail2ban/jail.local.example /etc/fail2ban/jail.local
```

## Adjust paths and whitelist

Edit `/etc/fail2ban/jail.local`:
- set `ignoreip` (office/VPN ranges)
- set correct `logpath` to the repository `logs/nginx/*.log`
- optionally switch `banaction` to `nftables-multiport` if needed

## Start and verify

```bash
sudo systemctl enable fail2ban
sudo systemctl restart fail2ban
sudo fail2ban-client status
sudo fail2ban-client status nginx-auth-bruteforce
sudo fail2ban-client status nginx-scanners
```

## Optional filter testing

```bash
sudo fail2ban-regex /opt/volt_store/logs/nginx/auth_access.log /etc/fail2ban/filter.d/nginx-auth-bruteforce.conf
sudo fail2ban-regex /opt/volt_store/logs/nginx/scanners_access.log /etc/fail2ban/filter.d/nginx-scanners.conf
```

## Notes

- Keep Nginx rate limiting enabled for auth endpoints.
- Fail2Ban should run on the host, not in a regular app container.
- If your host runs behind Cloudflare or another proxy, configure real client IP restoration first.
